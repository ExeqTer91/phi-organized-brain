Thank you for your interest in contributing to this research project!

## How to Contribute

### Reporting Issues

If you find bugs or have suggestions:

1. Check if the issue already exists
2. Create a new issue with a clear description
3. Include relevant data, error messages, and steps to reproduce

### Code Contributions

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Run tests to ensure everything works
5. Commit with clear messages
6. Push and create a Pull Request

### Code Style

- Follow PEP 8 for Python code
- Use meaningful variable names
- Add comments for complex logic
- Include docstrings for functions

### Data Contributions

If you have EEG datasets that could help validate the findings:

1. Ensure data is properly anonymized
2. Include metadata (sampling rate, channel count, paradigm)
3. Describe the preprocessing pipeline
